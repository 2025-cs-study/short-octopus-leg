# Part 1 | AI 활용의 첫걸음

## 1. 바이브 코딩이란?

자연어로 의도와 맥락을 전달하면 AI가 구현, 수정, 리팩토링까지 수행하는 개발 방식.
- 안드레이 카파시(Andrej Karpathy)가 제안
- 구현 세부사항보다 "무엇을 만들고 싶은지" 전달
- AI를 단순 도구가 아닌 페어 프로그래머로 활용

### 기존 프로그래밍 vs 바이브 코딩

```plaintext
// 기존 방식
개발자가 직접 코드 작성 → 디버깅 → 테스트

// 바이브 코딩
자연어로 설명 → AI가 코드 생성 → 검토 및 수정
```

### 당근마켓의 사용 예시

#### 1) AI 프로젝트 캐처 슬랙 봇

**배경**:
- 여러 팀에서 다양한 AI 실험 진행 중
- 정보가 공유되지 않아 중복 작업 발생
- 협업 기회를 놓치는 문제

**개발 과정**:
```plaintext
1단계: 아이디어 제안
Kyle (CEO Staff) → 슬랙에 아이디어 공유
"AI 관련 논의를 자동으로 수집하고 정리하면 어떨까?"

2단계: 빠른 구현 (1일)
Alden (엔지니어) → Cursor + 프롬프트 스튜디오 활용
- 슬랙 메시지 중 AI 프로젝트 감지
- 태스크, 카테고리, 요약 자동 생성
- 전용 채널에 자동 등록

3단계: 비개발자의 직접 개선
Kyle → 프롬프트 스튜디오로 분류 기준 직접 수정
"코딩 몰라도 기능 개발에 기여할 수 있다는 확신"
```

**결과**:
- 개발 기간: **하루**
- 비개발자도 분류 로직 직접 수정
- 전사 AI 프로젝트 정보 통합 관리

#### 2) Ratiosnap 피그마 플러그인

**과정**:
```plaintext
문제 인식:
"디자인 작업할 때 비율 맞추는 게 너무 비효율적이야"

Cursor에게 설명:
"이런 작업 방식이 불편한데, 해결 방법 제안해줘"

AI 응답:
- 문제 분석 완료
- 기능 요구사항 정리 (PRD처럼)
- UI 구성안 제시
- 코드 작성

Hazel의 역할:
- AI 제안 검토
- 디자인 감각 더하기
- 필요한 부분 피드백
```

**결과**: 30분 만에 실제 사용 가능한 피그마 플러그인 완성

## 2. 바이브 코딩 툴

### 1) Cursor

**특징**:
- VS Code 기반이라 기존 개발자에게 익숙함
- 전체 프로젝트 맥락을 이해하고 코드 생성
- 여러 파일을 동시에 수정 가능

**주요 기능**:

```plaintext
1. Chat (Cmd+L)
- AI와 대화하며 코드 질문
- 코드 설명 요청
- 구현 방법 논의

2. Composer (Cmd+I)
- 여러 파일 동시 수정
- 프로젝트 전체 구조 변경
- 대규모 리팩토링

3. Cmd+K
- 선택한 코드 바로 수정
- 인라인으로 빠른 변경
- 주석 → 코드 자동 생성
```

### 2) 프롬프트 스튜디오

**활용 사례**:
- 슬랙 메시지 자동 분류
- 데이터 분석 자동화
- 간단한 봇 제작

**장점**:
- 프롬프트만으로 LLM 기능 구현
- 비개발자도 쉽게 사용
- 빠른 프로토타입 제작

### 3) 기타 도구

```plaintext
Claude Code
- 터미널 기반 AI 코딩 도구
- 명령어로 파일 생성/수정

GitHub Copilot
- VS Code 확장 프로그램
- 코드 자동완성에 특화

Windsurf
- Cursor 대안
- 유사한 기능 제공
```

## 3. 바이브 코딩 프롬프트 입력 방법

### 효과적인 프롬프트 구조

```plaintext
[기술 스택] + [기능 설명] + [세부 요구사항] + [제약 조건]
```

**예시**:
```plaintext
❌ 나쁜 예
"로그인 만들어줘"
→ 너무 모호함, AI가 뭘 만들어야 할지 불명확

⭕ 좋은 예
"React로 로그인 폼을 만들어줘.
- 이메일, 비밀번호 입력 필드
- 이메일 형식 검증 (정규식 사용)
- 비밀번호 8자 이상, 영문+숫자 조합 확인
- 에러 메시지는 각 필드 아래 빨간색으로 표시
- Tailwind CSS로 스타일링
- 제출 시 /api/login으로 POST 요청"
→ 구체적이고 명확함
```

### 프롬프트 작성 4단계

**1단계: 컨텍스트 제공**
```plaintext
"React 프로젝트에서, TypeScript를 사용해서"
"Express 백엔드 API에서, MongoDB와 연동해서"
```

**2단계: 핵심 기능 설명**
```plaintext
"사용자 프로필 편집 폼을 만들어줘"
"상품 목록을 페이지네이션과 함께 표시해줘"
```

**3단계: 세부 요구사항**
```plaintext
"- 이름, 이메일, 전화번호 필드 필요
- 각 필드는 실시간 검증
- 저장 버튼 클릭 시 API 호출
- 로딩 상태 표시"
```

**4단계: 제약 조건 및 스타일**
```plaintext
"- Material-UI 컴포넌트 사용
- 모바일 반응형으로 작성
- 접근성(a11y) 고려"
```

### 단계별 개발 프롬프트

#### 1단계: 핵심 기능부터

```plaintext
"채팅 UI를 만들어줘.
- 메시지 목록 표시
- 입력창과 전송 버튼
- 심플한 디자인"
```

#### 2단계: 기능 추가

```plaintext
"방금 만든 채팅 UI에 기능 추가해줘.
- 메시지에 타임스탬프 표시
- 내 메시지는 오른쪽 정렬
- 상대 메시지는 왼쪽 정렬
- 각각 다른 색상 배경"
```

#### 3단계: 리팩토링

```plaintext
"이 채팅 컴포넌트를 개선해줘.
- Message 컴포넌트로 분리
- 반복되는 스타일을 상수로 추출
- PropTypes 또는 TypeScript 타입 추가"
```

### 시나리오별 프롬프트 예시

#### API 연동

```plaintext

- 로딩 상태 관리 (로딩 중 스피너 표시)
- 에러 발생 시 에러 메시지 표시

```

#### 폼 검증

```plaintext
"회원가입 폼의 검증 로직을 추가해줘.
- 이메일: 이메일 형식 확인
- 비밀번호: 8자 이상, 영문+숫자+특수문자
- 비밀번호 확인: 비밀번호와 일치
```

## 4. 주의사항

### 장점

1. 빠른 프로토타입 개발
- 4시간 만에 게임 제작 가능
- 반복 작업 자동화
- 아이디어를 빠르게 검증

2. 진입 장벽 낮춤
- 비개발자도 간단한 도구 제작
- 새로운 기술 스택 빠르게 학습
- 예제 코드 즉시 생성

3. 생산성 향상
- 단순 코드 작성 시간 감소
- 복잡한 로직에 집중 가능
- 문서화 자동 생성

### 주의사항

1. 코드 품질 문제

```python
# AI가 생성한 코드의 문제점
def get_users():
    users = []
    # N+1 쿼리 문제 발생!
    for post in Post.query.all():
        user = User.query.get(post.user_id)
        users.append(user)
    return users

# 개선 필요
def get_users():
    # Join으로 최적화
    return db.session.query(User)\
        .join(Post)\
        .all()
```

2. 보안 취약점

```javascript
// 나쁜 예: AI가 민감 정보 하드코딩
const apiKey = "sk_live_abcd1234";
const dbPassword = "mypassword123";

// 좋은 예: 반드시 환경 변수로 수정
const apiKey = process.env.API_KEY;
const dbPassword = process.env.DB_PASSWORD;
```

3. 과도한 의존 위험
- 기본기 없이 AI만 믿으면 디버깅 불가
- 생성된 코드를 이해하지 못하면 유지보수 어려움
- 복잡한 버그 발생 시 해결 어려움

> 바이브 코딩은 강력한 도구지만, 기본기가 있어야 제대로 활용 가능. AI가 생성한 코드를 무조건 신뢰하지 말고 항상 검증하고 이해해야 함.
